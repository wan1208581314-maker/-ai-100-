# 创意发散 - 项目日志

## v2.2 — 弹簧物理与飘逸运动（2026-05-09）

### 新增功能

**弹簧物理拖拽（需求 #24）：**
- 拖拽父节点时，所有后代节点跟随运动
- 采用 lerp + 多频飘动方案，节点有"被风吹过"的飘逸感
- 每个节点独立参数：跟随速度、飘动频率、飘动幅度、相位各不相同
- 层级越深的节点飘动幅度越大、跟随越慢
- 使用 `transform: translate()` 移动节点，避免与 CSS 浮动动画冲突
- 拖拽结束后 `commitSpring()` 将最终位置写回数据模型

**关键实现细节：**
- `startSpring()` — 初始化时为每个后代节点生成独立的飘动参数（随机相位、频率、幅度）
- `updateSpring()` — 每次 mousemove 调用，lerp 跟随 + 三层 sin/cos 飘动叠加
- `commitSpring()` — mouseup 时清除 transform，用 left/top 定位
- `clearSpring()` — 清理弹簧状态，恢复动画

**CSS 处理：**
- `.spring-active` 类：`animation: none !important` 抑制 CSS 浮动动画
- 弹簧期间通过 `el.style.animation = 'none'` 内联样式强制禁用

### Bug 修复

- **折叠/展开 UI 缺失（需求 #25）：** 节点数量徽章改为可点击，点击切换折叠/展开状态。折叠时徽章显示"N ▸"灰色样式
- **撤销选中状态不同步：** `undo()` 中添加 `removeConfirmIcon()` 调用，子节点全部撤销时重置父节点 collapsed 状态
- **节点入场定位闪烁：** 使用固定 inline width/height（120px/90px）代替 getBoundingClientRect，避免动画期间尺寸错误

### Prompt 优化

**联想接口（/api/associate）：**
- System Prompt：从泛泛的"创意联想助手"改为强调强相关性
- User Prompt：8 个联想方向建议（工具/场景/上下游/风格/人物/痛点/搭配/趋势）
- 核心原则：每个词必须与输入词强相关，"看到就能理解为什么想到它"
- temperature: 0.7（比之前更聚焦）

**创意方案接口（/api/generate）：**
- 强调可落地产出：标题、核心概念、执行步骤、平台建议、目标人群
- temperature: 1.0（保持发散性）
