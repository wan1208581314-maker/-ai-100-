# 桌面宠物制作文档

> 本文档是一份完整的制作指南。你只需要准备一套角色动画素材，然后把本文档连同素材一起发给 Claude Code（或其他 AI），就能制作出一个可交互的桌面宠物程序。

---

## 一、你需要准备什么

### 1.1 角色动画素材（spritesheet）

你需要一张**序列帧图集**（spritesheet），格式为 `.webp` 或 `.png`。

这张图是一张大图，里面按**网格排列**了角色的所有动作帧。规则如下：

```
每一行 = 一个动作状态
每一格 = 一帧动画
所有格子大小相同（例如 192x208 像素）
```

示例：一张 1536x2288 的图集，按 192x208 切分，可以得到 8 列 x 11 行 = 88 帧。

### 1.2 动作清单（manifest.json）

你需要一个 JSON 文件，描述图集中每一行对应什么动作、有多少帧。格式如下：

```json
{
  "spritesheet": "spritesheet-11.webp",
  "spritesheetSize": [1536, 2288],
  "cellSize": [192, 208],
  "columns": 8,
  "rows": [
    {"row": 0, "state": "idle", "frames": 6, "description": "待机"},
    {"row": 1, "state": "running-right", "frames": 8, "description": "向右跑"},
    {"row": 2, "state": "running-left", "frames": 8, "description": "向左跑"},
    {"row": 3, "state": "waving", "frames": 4, "description": "挥手"},
    {"row": 4, "state": "jumping", "frames": 5, "description": "跳跃"},
    {"row": 5, "state": "failed", "frames": 8, "description": "失败"},
    {"row": 6, "state": "waiting", "frames": 6, "description": "等待"},
    {"row": 7, "state": "running", "frames": 6, "description": "忙碌"},
    {"row": 8, "state": "review", "frames": 6, "description": "通知"},
    {"row": 9, "state": "long-idle-angry-stomp", "frames": 8, "description": "久未互动生气跺脚"},
    {"row": 10, "state": "pet-shy-heart", "frames": 8, "description": "摸摸害羞"}
  ]
}
```

### 1.3 文件目录结构

把素材放在 `assets/` 文件夹下：

```
desktop-pet-app/
  pet.py               ← 主程序（AI 生成）
  assets/
    spritesheet-11.webp   ← 你的角色素材
    manifest.json         ← 动作清单
```

## 二、发给 AI 的完整 Prompt

复制以下内容，连同你的素材文件一起发给 Claude Code：

---

### Prompt 开始

```
我要做一个 Windows 桌面宠物程序。我的素材在 assets/ 目录下：

- assets/spritesheet-11.webp — 序列帧图集（角色动画素材）
- assets/manifest.json — 动作清单，描述每行的动作名和帧数

请根据 manifest.json 中的信息，用 Python + tkinter + Pillow 制作一个可交互的桌面宠物程序。

要求如下：

## 窗口
- 无边框透明悬浮窗，始终置顶
- 不在任务栏显示
- 背景色设为 #010101，用 tkinter 的 transparentcolor 透明掉
- 窗口大小 = 每帧大小（从 manifest.json 的 cellSize 读取）
- 默认出现在屏幕右下角
- 窗口位置自动保存，下次启动恢复

## 动画
- 读取 spritesheet 图集，按 cellSize 切分每一帧
- 启动时预加载所有帧到内存（用 PIL 裁切 + ImageTk.PhotoImage）
- 默认每 120ms 切换一帧
- 每个动作可以有独立的帧间隔（manifest.json 中的 interval 字段，可选）
- loop: true 的动作循环播放
- loop: false 的动作播完后自动切换到 next 字段指定的状态

## 状态机
- 根据 manifest.json 中的 rows 生成状态表
- 每个状态包含：row（行号）、frames（帧数）、loop（是否循环）、next（播完后跳转的状态，可选）、interval（帧间隔ms，可选）
- 状态切换时重置帧计数、更新画面、重启动画定时器
- 允许重复进入非循环状态（保证点击可以反复触发）

## 鼠标交互
- 鼠标悬停（Enter）：切换到 jumping
- 单击（释放时，非拖拽）：切换到 waving
- 双击：切换到 pet-shy-heart
- 拖拽：窗口跟随鼠标移动，根据水平移动方向切换 running-right / running-left，松开后回到 idle
- 右键：弹出菜单（唤醒、回到待机、回到右下角、暂停/继续、退出）

## 长时间未互动（30分钟）
- 30分钟没有任何鼠标操作
- 记录当前位置作为"角落"
- 自动计算屏幕正中央位置（水平居中、垂直居中，适配任意分辨率）
- 播放 running-right 或 running-left 跑到屏幕中间
- 到达中间后循环播放 long-idle-angry-stomp（一直跺脚生气）
- 如果用户点击她 → 切换到 pet-shy-heart 害羞
- 害羞播完后 → 播放 running-left 或 running-right 跑回之前记录的角落位置 → 回到 idle

## 害羞动画速度
- pet-shy-heart 的帧间隔设为 200ms（比默认 120ms 慢，让害羞动作更明显）

## 技术栈
- Python 3.12+
- tkinter（Python 内置）
- Pillow（pip install Pillow）
- ctypes（用于 Windows 隐藏任务栏图标）

## 文件结构
- pet.py — 主程序单文件
- assets/spritesheet-11.webp — 素材
- assets/manifest.json — 动作清单

请生成完整的 pet.py 代码。
```

### Prompt 结束

## 三、制作过程详解

### 第一步：确认素材

AI 会先读取你的 `manifest.json`，确认：
- 图集尺寸（spritesheetSize）
- 每帧大小（cellSize）
- 行数和每行的动作名、帧数
- 是否有 loop / next / interval 等配置

### 第二步：生成主程序

AI 会生成一个 `pet.py` 文件，包含以下核心模块：

#### 3.2.1 窗口创建
```python

root.overrideredirect(True)           # 无边框
root.attributes("-topmost", True)     # 置顶
root.attributes("-transparentcolor", "#010101")  # 透明
root.config(bg="#010101")
```

#### 3.2.2 隐藏任务栏
```python
hwnd = ctypes.windll.user32.GetParent(root.winfo_id())
style = ctypes.windll.user32.GetWindowLongW(hwnd, -20)

ctypes.windll.user32.SetWindowLongW(hwnd, -20, style)
```

#### 3.2.3 素材加载

## 四、可自定义的参数

在 `pet.py` 顶部可以调整以下参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `FRAME_INTERVAL` | 120ms | 默认动画帧间隔 |
| `IDLE_TIMEOUT_MS` | 30分钟 | 多久没互动触发生气 |
| `pet-shy-heart` 的 `interval` | 200ms | 害羞动画速度（越大越慢） |
| `#010101` | 透明色 | 背景透明的颜色值，需和素材背景不冲突 |

## 六、素材制作建议

如果你还没有素材，可以用 AI 图片生成工具制作序列帧：

1. 先生成一张角色基础站姿（idle 的第一帧）
2. 以这张图为参考，逐个生成每个动作的序列帧
3. 帧之间保持角色位置、大小、风格一致
4. 用图片编辑工具（如 Photoshop、Figma）把所有帧拼成一张大图
5. 背景必须是透明的（PNG 或 WebP 格式）
6. 每帧大小必须一致

动作建议（最少 11 个）：

| 行号 | 动作 | 帧数 | 用途 |

## 七、常见问题

### Q: 透明窗口有黑底/白底？
确保素材本身是透明背景（PNG/WebP），且 `transparentcolor` 值和 Canvas 的 `bg` 一致。

### Q: 角色有锯齿？
素材建议用 WebP 格式，分辨率至少 192x208 每帧。

### Q: 想加更多动作？
在 manifest.json 中增加行，在 STATES 字典中增加对应状态即可。

### Q: 想改触发时间？
修改 `IDLE_TIMEOUT_MS` 的值（单位毫秒）。

### Q: 想打包成 exe？
用 PyInstaller：`pip install pyinstaller && pyinstaller --onefile --windowed --add-data "assets:assets" pet.py`
